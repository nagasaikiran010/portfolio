01
Static & Dynamic

MALWARE ANALYSIS

Malware analysis is about understanding the enemy's weapons. I'm fascinated by how attackers craft malicious software — from simple droppers to sophisticated RATs and ransomware. Understanding how malware works from the inside is the best way to build defences against it.

I approach this from both the static side (reading disassembly, strings, imports) and the dynamic side (running samples in sandboxes, watching syscalls, network behaviour, registry changes).

02
PWN / CTF

BINARY EXPLOITATION

Binary exploitation is where security meets low-level engineering. I love the puzzle-like nature of finding and chaining together vulnerabilities in compiled binaries — stack overflows, heap corruption, format strings, and ROP chains.

I practise primarily through CTF competitions and platforms like pwn.college and picoCTF. The process of turning a crash into a shell is one of the most satisfying experiences in offensive security.

03
IDA / Ghidra

REVERSE ENGINEERING

Reverse engineering is the art of reading what a machine says without having the source. I'm drawn to it because it demands patience, pattern recognition, and a deep understanding of how programs actually execute — not how developers say they do.

From crackmes to firmware analysis, RE is an essential skill that bridges malware analysis, vulnerability research, and binary exploitation.

04
Web / API

BUG BOUNTY

Bug bounty is where curiosity meets real-world impact. Unlike CTFs, bug bounty targets are live production systems used by real people — which makes finding vulnerabilities both more challenging and more meaningful.

I focus primarily on web and API vulnerabilities — IDOR, authentication flaws, business logic bugs, and injection vulnerabilities. My work here has led to real recognition.
